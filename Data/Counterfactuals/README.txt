Contents of $PROJDIR/Data/Counterfactuals
-----------------------------------------


Data files for query types: ADD_LINK, CHANGE_LINK, DELETE_LINK

Derived from: DrugMechDB


File name format: {QUERY_TYPE}_{POSNEG}_{LINK_TYPE}_r{COUNT}.json

QUERY_TYPE: Indicates type of query
	AddLink: ADD_LINK
	change: CHANGE_LINK (aka Invert-Link)
	delete: DELETE_LINK

POSNEG: Indicates whether they are +ive or -ive samples
	pos: +ive
	neg: -ive

LINK_TYPE: Indicates what type of link involved in the counterfactual
	dpi: Drug-Protein interaction (aka Surface counterfactual)
	ppi: InternalNode-Protein interaction
		Indicates that an internal link (aka Deep counterfactual) is involved, i.e. a link NOT from the Drug node.
		InternalNode is usually Protein, but can also be: ChemicalSubstance, GeneFamily


General file format:
Each is a JSON file containing a list of samples. Each sample represents one query, and is a serialized representation of
	`drugmechcf.llmx.test_addlink.AddLinkTask` or `drugmechcf.llmx.test_editlink.EditLinkTask`.
See notes below for how instances of those classes are created.
